§ 01Purpose

This notebook is a sibling of 02_eval_no_ground_truth.py. It adds a fourth signal layer that cross-checks ai_extract field-level confidence against ai_parse_document element-level confidence over the cited bounding-box region. The added layer catches a failure mode that neither score alone can flag:

The model is confident over OCR garbage. extract_conf is high because the extractor reasoned cleanly over what it saw, but the source element was mis-OCR'd in the first place — so the value is wrong with high confidence.

The three layers from the original notebook 02 are unchanged. See docs/notebook-02-methods.md for those, and docs/plan-cross-confidence-check.md for the original design proposal.

§ 02Inputs and outputs

Inputs
  • deeds_extracted_flat — extraction output, per-field extract_conf, citation_ids.
  • deeds_parsed_elements — per-element text, bbox, confidence.
Outputs (suffixed _xconf)
  • MLflow run under deeds_poc_eval_no_gt_xconf.
  • deeds_review_flags_xconf — per-doc with cross-conf columns.
  • deeds_field_review_xconf — per-(doc, field) review queue.

§ 03Three deterministic signal layers

All three are pure logic, statistics, or geometry — no per-document LLM API cost.

Layer 1 · parse-side
Doc-level corpus outliers

Four booleans from ai_parse_document aggregates: conf_mean, conf_stddev, worst_page_mean, error_status_count.

Catches: "this whole doc is OCR'd badly".

Layer 2 · extract-side
Field-level extract confidence

Per-cell threshold on <field>_extract_conf with non-null guard. Rolls up to low_conf_field_count, low_conf_fields, min_extract_conf.

Catches: "this specific cell is uncertain".

Layer 3 · cross (new)
Bbox-overlap fusion

ai_extract cited bbox ↔ ai_parse_document element confidence via IoU. Produces source_conf and three calibration booleans.

Catches: "model is confident over OCR garbage".

The earlier absence-aware LLM judge was removed for cost. The cross-confidence layer subsumes its main signal — calibration mismatch — at the field level, deterministically.

§ 05Cross-confidence pipeline (§3c)

Six-step vectorized PySpark pipeline.

Diagnostic<field>_citation_unmatched is set when a field has non-empty citation_ids but no parsed element overlaps any cited bbox (source_conf IS NULL). Surfaced for triage; not rolled into review_priority — usually means the extractor hallucinated a region.

§ 06Calibration-mismatch flags (§3d)

Three booleans per (doc, field), only when both extract_conf and source_conf are non-null.

Roll-ups: calibration_mismatch_field_count (0–6) and min_source_conf — single sortable signal per document.

§ 07Output tables

7.1 deeds_review_flags_xconf (per document)

The review_priority formula gains one term:

review_priority =
    outlier_low_mean                       (0–1)
  + outlier_high_variance                  (0–1)
  + outlier_bad_page                       (0–1)
  + has_parse_error                        (0–1)
  + !extraction_shape_ok                   (0–1)
  + low_conf_field_count                   (0–6)
  + calibration_mismatch_field_count       (0–6)   ← NEW

New range 0–17. The per-field cross-confidence columns (<field>_source_conf, three derived booleans, <field>_citation_unmatched) are projected onto the table for cell-level inspection without joining back to the long-format table.

7.2 deeds_field_review_xconf (per cell)

Inclusion rule: row is included if any of <field>_low_extract_conf, the field's shape check failed, or <field>_calibration_mismatch is true.

Per-row schema gains: source_conf DOUBLE, calibration_mismatch BOOLEAN, model_uncertain_clean_source BOOLEAN, consistent_low_source BOOLEAN, citation_unmatched BOOLEAN. The reasons array can now contain "calibration_mismatch", "model_uncertain_clean_source", "consistent_low_source".

Display ordering: calibration_mismatch DESC, extract_conf ASC, image_name — confident-over-garbage cells float to the top.

§ 08Reviewer workflow

  1. Open deeds_review_flags_xconf and sort review_priority DESC. Top documents are highest priority.
  2. Inspect min_source_conf and calibration_mismatch_field_count to gauge how much of the priority is cross-confidence-driven.
  3. Open deeds_field_review_xconf, filter on image_name. Default sort surfaces calibration_mismatch = TRUE rows first.
  4. Use cited_regions[*].coord and page_id to jump to the cited bbox on the source page. Compare model value, OCR'd text, and the actual scan.
  5. model_uncertain_clean_source cells flag a different remediation path — clean OCR but a hesitant model. Usually a schema or instruction issue, not a parsing one.
  6. citation_unmatched = TRUE is rare and usually means the extractor cited a region the parser did not produce. Investigate directly.

§ 09Tunable knobs

§ 10Failure modes to watch

  • source_conf all null after §3c. The citation VARIANT decode failed or the bbox VARIANT decode failed. §3c prints a sample of each at the top. Most common cause: bbox is in [x, y, w, h] form rather than [x0, y0, x1, y1]. The IoU UDF normalizes axis order but not parameterization — _bbox_to_array needs a third fallback.
  • Massive citation_unmatched count. Either IoU threshold is too high or page_id is mismatched. Lower IOU_THRESHOLD to 0.0 as a diagnostic.
  • calibration_mismatch_field_count always 0. Thresholds too tight, or corpus has no confident-over-garbage cells. Raise CALIBRATION_MISMATCH_SOURCE_CONF to 0.7.
  • IoU UDF asserts fail at notebook start. Math regression. Do not modify the unit-test values — fix iou_udf instead. The asserts are the contract.

§ 11Verification

  1. The IoU UDF asserts in §3c run on every execution and must pass.
  2. The sample VARIANT print in §3c shows the live structure of citations and bbox. Confirm both decode to non-null arrays of four numbers.
  3. End-to-end smoke on the 6-doc POC. Expected pattern: clean PDFs get min_source_conf close to 1.0; scanned TIFFs show lower min_source_conf, especially on BookNumberType and PageNumber. At least one cell with non-empty citation_ids should resolve to a non-null source_conf.
  4. GT-mode cross-check (optional). After running notebook 03, join deeds_extracted_vs_gt to deeds_field_review_xconf on (image_name, field) and check whether calibration_mismatch = TRUE rows skew toward classification IN ('FP', 'FN', 'FP_FN') rather than TP / TN. If yes, the cross-check is meaningful and the threshold defaults are good.
