§ 01Two confidences, two granularities

The pipeline never reads parse confidence and extract confidence as the same thing. Different functions, different objects, different failure modes.

Parse confidence
ai_parse_document v2.0

One score per parsed element — line, span, region. Answers "how well did we read the document?"

Aggregations: per-doc mean, stddev, worst-page mean, error count.

Extract confidence
ai_extract v2.1

One score per extracted field. Answers "how sure is the model about this field?"

Aggregations: per-field threshold + per-doc roll-up count.

Failure mode neither catches alone Confident over OCR garbage. The extractor reasoned cleanly over what it saw, but the parser mis-OCR'd the source. extract_conf stays high; parse confidence on the cited bbox is low. Catching this is the job of the third statistic, cross-confidence (§ 04).

§ 02Doc-level corpus outliers

Four parse-side booleans per document, computed against the current run's corpus. Corpus-relative rather than against a persisted baseline, so each batch is self-describing.

FlagRuleReading
outlier_low_meanconf_mean ≤ corpus_p10Doc is in the bottom 10% of parse confidence.
outlier_high_varianceconf_stddev > Q3 + 1.5·IQRTukey upper fence on stddev — pages disagree.
outlier_bad_pageworst_page_mean ≤ corpus_p10One page is unusually bad even if the doc average isn't.
has_parse_errorerror_status_count > 0Parser raised a structured error.
Why IQR, not z-scoreOCR confidence-stddev distributions aren't normal — Tukey fences are robust to that.

§ 03Field-level cutoff

One threshold per field cell; one roll-up per doc.

<field>_low_extract_conf =
    value IS NOT NULL
    AND extract_conf IS NOT NULL
    AND extract_conf < LOW_EXTRACT_CONF_THRESHOLD     -- default 0.7

Per-doc roll-ups:

  • low_conf_field_count (int 0–6).
  • low_conf_fields (array of strings) — which ones.
  • min_extract_conf — single sortable signal; null confidences coalesce to 1.0.
InvariantEmpty fields are not failures. Shape rules and field-level flags only fire on non-null values — a blank value is correct when the entity isn't in the document.

§ 04Cross-confidence (xconf sibling notebook)

The two streams cross at the cited bbox. ai_extract returns citation_ids for each field; those resolve to bboxes through metadata.citations. ai_parse_document exposes element confidence + bbox. The pipeline joins them via IoU and produces <field>_source_conf — the minimum parse-side element confidence inside the cited region.

§ 05Composite — review_priority

Multiple signals, one sortable integer. Field-level signals dominate by design so a document with multiple suspicious cells outranks one with a single doc-level flag — matches how a human triages.

review_priority =
    outlier_low_mean                       (0–1)
  + outlier_high_variance                  (0–1)
  + outlier_bad_page                       (0–1)
  + has_parse_error                        (0–1)
  + !extraction_shape_ok                   (0–1)
  + low_conf_field_count                   (0–6)
  + calibration_mismatch_field_count       (0–6)    [xconf only]
  + format_violation_count                 (0–6)    [production only]
  + no_citation_field_count                (0–6)    [production only, gated]

Range depends on which notebook you ran:

  • Notebook 02 (baseline) — 0–11. First six terms.
  • Notebook 02 (xconf) — 0–17. Adds calibration_mismatch_field_count.
  • Production analytics — 0–24. Adds format_violation_count and no_citation_field_count.

§ 06How it lands in MLflow

Every statistic is exposed as a @scorer. Scorers are pass-throughs around precomputed values — no model-as-judge calls, no recomputation. mlflow.genai.evaluate is being used as a structured logging vehicle for deterministic signals, not as a judge harness.

§ 07The reading order

Four moves — no software between you and the source page.

Move 1
Open the MLflow run

Top-line scorers tell you whether this batch is good in aggregate. Sort runs by min_source_conf or calibration_mismatch_field_count when comparing schema / instruction versions.

Move 2
Pivot to the per-doc Delta

deeds_review_flags (or _xconf / _dab variant), sorted review_priority DESC. One row per document with every flag projected on.

Move 3
Filter the per-cell queue

deeds_field_review, filter on the document. Default sort surfaces the highest-priority cells: calibration_mismatch DESC, extract_conf ASC.

Move 4
Jump to the cited bbox

Use cited_regions[*].coord + page_id to land on the right region of the source page. Compare model value, OCR'd text, and the actual scan.

Discipline The pipeline flags. Nothing learns. Reviewer feedback doesn't write back into thresholds or scorers — that would couple the offline eval to the production loop and erase the deterministic guarantee. Threshold changes live in code, gated by inspection of the joint distribution.
