Notebook 01 · Parse and Extract · Workflow

From scanned deed pages to a clean extraction table

Runs ai_parse_document v2.0 over a UC volume of scanned deeds, captures per-element / per-page / per-doc confidence as descriptive signals, then runs ai_extract v2.1 on the parsed VARIANT to populate the six-field header schema. Nothing is filtered — every row is preserved, including parse errors.

§ 01Purpose

Batch notebook that walks a UC volume of scanned U.S. real-estate deed (DED) documents and produces four Delta tables ready for the downstream notebooks (02 = eval-no-GT, 03 = eval-with-GT, plus the production xconf / batch analytics). Every confidence signal it captures is descriptive — used to rank documents and elements, never to filter them out.

Rule: all rows are retained, including those with non-null error_status. Confidence and structure are kept inside the VARIANT for downstream consumers.

§ 02Inputs and outputs

Input
UC volume

/Volumes/fins_genai/unstructured_documents/deeds/

Scanned deed pages. Glob filter: *.{pdf,PDF,tif,TIF,tiff,TIFF,jpg,jpeg,png}. Recursive lookup.

Requires DBR 17.1+ for ai_parse_document, or serverless SQL.

Outputs · 4 Delta tables
  • deeds_parsed — raw ai_parse_document VARIANT per file.
  • deeds_parsed_elements — exploded elements with confidence, bbox, conf_tier.
  • deeds_parsed_pages — per-page confidence stats.
  • deeds_extracted_flat — one row per doc with 6 extracted fields + per-field extract_conf, citation_ids, and doc-level confidence stats.

§ 03Parse stage — ai_parse_document

Reads the volume as binaryFile, then calls ai_parse_document v2.0 with descriptionElementTypes='*' and an imageOutputPath so each page is rendered as a PNG into a sibling volume folder. The rendered images are how downstream notebooks overlay citation bboxes for reviewer triage. 

ai_parse_document(
    content,
    map(
        'version', '2.0',
        'imageOutputPath', '/Volumes/fins_genai/unstructured_documents/deeds/images/',
        'descriptionElementTypes', '*'
    )
)

The image_name column — the filename stem without extension — becomes the canonical join key across every downstream table.

§ 04Confidence tiers — three independent signals

OCR confidence is left-skewed (heavy mass near 1.0), so a pure z-score makes a poor global threshold. The conf_tier column combines three complementary signals — any one firing flags the element as low.

Signal 1 · Absolute floor
Universally bad elements

confidence < 0.5

Catches the truly broken regardless of corpus shape.

Signal 2 · Corpus percentile
Worst slice of this run

confidence < corpus_p10

Distribution-agnostic. Always flags ~10% of elements.

Signal 3 · Doc-internal z-score
Local anomalies

doc_z < −1.5

Catches outliers in otherwise-clean docs.

The raw corpus_z and doc_z values are kept on every row as diagnostic columns for ad-hoc analysis, even though the tier no longer hinges on corpus_z alone (skewed distributions made it unreliable as a gate).

§ 05Per-page stats — deeds_parsed_pages

Group deeds_parsed_elements by (image_name, page_id) and aggregate confidence statistics:

n_elements, conf_mean, conf_stddev, conf_min, conf_max

The page table is the join target for the worst-page roll-up below — and the place reviewers go when they want to know which page of a multi-page doc is the bad one.

§ 06Per-doc stats

Two stages, joined together to produce a single doc-level row (held in memory as doc_stats, later joined into deeds_extracted_flat):

  • Doc-level distribution stats over the exploded elements: counts, mean, stddev, percentiles (p10, p25, p50, p75, p90), conf_iqr, low_conf_pct.
  • Per-page roll-ups: worst_page_mean, max_page_stddev, plus an argmin join that resolves worst_page_id.
  • error_status_count read from the raw VARIANT.
Why argmin via joinPySpark has no native argmin window. Filter page.conf_mean == roll-up.worst_page_mean and pick min(page_id) as the deterministic tiebreak — so reviewers always land on the first occurrence of the worst-page tie.

§ 07Extract stage — ai_extract v2.1

v2.1 enables two options the v2.0 path didn't have. Both run on the parsed VARIANT, not on raw text:

enableConfidenceScores
Per-field confidence

Each scalar leaf is wrapped as {value, confidence_score}. Score is in [0, 1].

enableCitations
Per-field citation IDs + doc-level bboxes

Each field returns citation_ids; the document carries metadata.citations (bbox + page_id) and metadata.pages.image_uri.

ai_extract(
    parsed,                                         -- the parsed VARIANT, not raw text
    '<EXTRACT_SCHEMA JSON>',
    map(
        'version',                '2.1',
        'enableConfidenceScores', 'true',
        'enableCitations',        'true',
        'instructions',           '<EXTRACT_INSTRUCTIONS>'
    )
)

Each scalar field is then projected to three columns by _field_select_exprs:

<field>                      -- value (STRING)
<field>_extract_conf         -- confidence_score (DOUBLE)
<field>_citation_ids         -- ARRAY<INT>

§ 08The six-field extraction schema

Each field has a written description that constrains the model. Strict instructions: use only what is printed; return null for absent fields; RecordingDate must be 8 digits YYYYMMDD.

String · free-form
DocumentTitle

Heading near top of first page. e.g. WARRANTY DEED, CORRECTION QUITCLAIM DEED. Verbatim. No boilerplate.

String · 1–3 chars
BookNumberType

Prefix label for book number. e.g. R, BK, OR, DR. Null if absent.

String · digits
BookNumberParsed

Book / volume number near the recording stamp. Null if filed by document number only.

String · digits or range
PageNumber

Page or range inside the book. e.g. 995, 0839, 2085-2087. Preserve leading zeros.

String · instrument ID
DocumentNumber

Unique recorder-assigned ID. Preserve leading zeros. Distinct from book/page.

String · YYYYMMDD
RecordingDate

Recording stamp date — not signing or notarization. Must be 8 digits.

§ 11Tunable knobs

KnobDefaultWhen to change
ABS_FLOOR0.5Lower if conf_tier = low never fires from this signal. Raise if it fires on everything.
CORPUS_PERCENTILE0.10This is the share of elements pulled into low by signal 2. Raise to 0.15 for a wider net; lower to 0.05 for top-decile-only triage.
DOC_Z_THRESHOLD−1.5More negative ⇒ stricter (only fires on heavy outliers). Less negative ⇒ more borderline elements surface.
imageOutputPathvolume images/Drop only if the downstream review UI doesn't need rendered page images. Citations carry image_uri from this folder.
EXTRACT_REVIEW_THRESHOLD0.7Plot-only reference line; must match LOW_EXTRACT_CONF_THRESHOLD in notebook 02 to stay coherent.

§ 12Failure modes and smoke checks

Section 8 of the notebook asserts the join-shape invariants:

assert n_parsed == n_extracted          # row count parity across the pipeline
assert n_elements > 0                   # ai_parse_document actually returned elements
  • Row-count mismatch between deeds_parsed and deeds_extracted_flat. Usually means ai_extract errored on at least one doc. extraction_error_message on the relevant rows tells you which.
  • deeds_parsed_elements empty. The VARIANT path parsed:document:elements didn't resolve. Inspect the raw parsed column structure; ai_parse_document may have returned an error structure instead of a document.
  • No low tier elements at all. Corpus is unusually clean — or the three thresholds are too lenient. Inspect conf_min and the flag_* columns directly.
  • All elements low. Corpus is uniformly bad or ABS_FLOOR / CORPUS_PERCENTILE are too aggressive. Raise ABS_FLOOR first; it's the absolute reference.
  • Per-field extract_conf all null. ai_extract v2.1 options didn't apply — verify the map(...) options are correct, including the version string '2.1' (not '2.0').
